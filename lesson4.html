Головна / Реалізація гри "Морський бій"

Алгоритм гри

Браузер розміщує кораблі, а ви намагаєтеся знайти і знищити їх. Завдання: потопити кораблі браузера за найменшу кількість ходів. В залежності від того наскільки добре вдається це зробити, ви отримуєте оцінку. В майбутньому гра буде виглядати як на малюнку 4.jpg. Замість таблиці 7 на 7 як на малюнку, починаємо з таблиці 1 на 7 і корабель буде один.

Проектування гри

  1. Користувач запускає гру
  2. А. Гра розміще корабель у випадку із позиції таблиці.
  3. Починається бій
  4. Повторювати наступні дії поки корабель не буде потоплений:
    1. А. Запитати у користувача кординати пострілу ("2","0" і тд).
    2. Б. Порівняти кординати з положенням і визначити результат: попадання, промах, корабель потоплений.
  6. Гра закінчена
  7. Вивести оцінку, яка визначається за кількістю спроб.

Корабель буде розташовуватися в трьох суміжних клітинках.

  • Для отримання даних від користувача будемо використовувати функцію prompt
  • Для виведення - функцію alert.

Псевдокод гри

Псевдокод — проміжна стадія між кодом Javascript і описанням програми на розмовній мові.

У псевдокоді спрощеного варіанта морського бою і розділ описання змінних розділ описання логіки програми.

  • Оголосити 3 змінні для зберігання позиції кожної клітинки корабля (location1, location2, location3).
  • Оголосити зміну для номеру поточної спроби (guess).
  • Оголосити зміну для кількості влучень (hits) і ініціалізувати їй значення 0.
  • Оголосити зміну для кількість спроб (guesses) і ініціалізувати їй значення 0.
  • Оголосити зміну для зберігання інформації про те, чи є потопленим в корабель чи ні (isSunk), і ініціалізувати їй значення False.
цикл: поки корабель не потоплений  отримати кординати постріли від користувача  порівняти дані з діапазоном вірних значень  якщо введені дані невірні,    вивести повідомлення з пропозицію ввести вірне число  інакше    додати 1 до guesses    якщо постріл влучив в корабель      додати 1 до кількості влучень      якщо кількість влучень рівна 3        присвоїти isSunk значення true        вивести повідомлення "Корабель затонув!"      кінець якщо    кінець якщо  кінець інакшекінець циклу

Програмування гри

Оголосити змінні 

var location1 = 3;var location2 = 4;var location3 = 5;var guess;var hits = 0;var guesses = 0;var isSunk = false;

Цикл: поки корабель не потонув отримати координати пострілу 

while (isSunk == false) {  guess = prompt("Ready aim fire! (enter a number 0-6):");}

Вбудована функція prompt призначена для отримання даних від користувача. Як і функція alert, prompt викликає на екран діалогове вікно з переданим рядком, але надає місце, в якому користувач може ввести відповідь. Ця відповідь у вигляді рядку повертається як результат виклику функції. Якщо користувач закрив діалогове вікно або нічого не відповів, функція повертає null ("Немає значення").

Виконуємо перевірку коректності введеного значення i якщо все вірно, збільшимо змінну guesses. 

if (guess < 0 || guess > 6) {  alert("Please enter a valid cell number!");} else {  guesses = guesses + 1;}

JavaScript автоматично перетворює рядок даних введений в prompt в число щоб порівняти з 0 і 6.

Перевіримо влучність: 

if (guess == location1) {  hits = hits + 1;} else if (guess == location2) {  hits = hits + 1;} else if (guess == location3) {  hits = hits + 1;}

Перевіримо чи досягла кількість влучень трьох 

if (hits == 3) {  isSunk = true;  alert("You sank my battleship!");}

Виведення даних після гри

Коли isSunk == true, цикл while зупиниться. Залишається вивести інформацію про те, наскільки добре користувач впорався зі своєю задачею. 

var stats = "you took" + guesses + "guesses to sink the battleship," + "which means your shooting accuracy was " + (3 / guesses);alert(stats);

У псевдокоді ми не повідомляємо користувачу влучив він в корабель (HIT) чи промахнувся (MISS).
