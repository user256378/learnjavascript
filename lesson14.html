Головна / Порівняння значень

Як порівнювати змінні

Якщо обидва значення відносяться до одного типу ми просто порівнюємо їх. Якщо значення відносяться до різних типів, ми перетворюємо їх до одного типу, а потім порівнюємо.

Порівняння числа з рядком 

99 == "99";// true99 == 99;// true99 == "vanilla";// false99 == NaN;// false

Порівняння булевого значення з іншим типом: в цьому випадку булеве значення перетворюється в число і два числа порівнюються (true -> 1, false -> 0):

1 == true;// true1 == 1;// true"1" == true;// true"1" == 1;// true

Порівняння null з undefined. Порівняння цих значень дає результат true:

undefined == null;// true

Перевірте, що буде результатом порівняння:

1 == " ";

Строга рівність (===) (оператор тотожності)

Два значення вважаються строго рівними лише в тому випадку, якщо вони відносяться до одного типу і містять одне значення.

Це означає, що якщо два значення відносяться до одного типу, ми переходимо до їх порівняння. Якщо ні — результат false. 

99 == "ninety-nine";// falsetrue == "true";// false
99 <= "100";// true

Щоб визначити результат порівняння "banana" < "mango" , треба використати алфавітний порядок. Отже "banana" < "mango" , так як "b" передує "m" . А "mango" < "melon" тому, що перші букви співпадають, але друга буква "a" передує "e" . В той же час "Mango" < "mango" , бо в порядку символів Юнікода "M" іде раніше за "m" .

Також існує оператор !== (строга нерівність).

Ще більше перетворень типів

При додаванні числа до рядка (і навпаки) Javascript перетворює число в рядок і виконує конкатенацію (поведінка протилежна тому, що відбувається у перевірці рівності): 

var addi = 3 + "4";// "34"var plusi = "4" + 3;// "43"

Що стосується інших арифметичних операторів: множення, ділення і віднімання, — Javascript розглядає їх як арифметичні, а не рядкові операції: 

var multi = 3 * "4";// 12var divi = 80 / "10";// 8var mini = "10" - 5;// 5
var order = 1 + 2 + "pizzas";// "3 pizzas"

Вираз опрацьовується зліва направо. 

var order = (1 + 2) + "pizzas";// "3 pizzas"
var order = 1 + (2 + "pizzas");// "12 pizzas"

Рівність об’єктів

При порівнянні об’єктів не важливо як порівнювати (== чи ===). Порівнюються посилання на ці об’єкти.

Два посилання рівні лише в тому випадку, якщо вони посилаються на один об’єкт.

Псевдоістина та псевдохиба

В Javascript існують значення, які не є ні true, ні false, але при цьому інтерпретуються як true або false в умовних виразах. Такі вирази називають «псевдоістиною» і «псевдохибою».

Правило:

Cпочатку визначте, що вважається псевдохибою, все інше буде псевдоістиною.

Псевдохибні значення: undefined, null, 0, " ", NaN.

Рядок як примітив і як об’єкт

В Javascript рядок виглядає і як примітив і як об’єкт (з підтримкою корисних методів опрацювання рядків).

Javascript створює рядки-об’єкти за нас при необхідності (коли для рядку викликається метод).
